Title: Round 1  (View)

For Round 1, we used a modified version of the eApplications form to test how caseworkers identify cases. Specifically, we were interested in knowing the following:
  • What data caseworkers need in order to identify the candidate cases (Change of Address (COA) and Severance (SEV))
  • What data is commonly missing if any
  • What does the identification process look like
  • What data is vital for identification (can go broader than candidates)

Title: Round 2 (COA)  (View)

Changes:
  • Workflow style bar is used to display information about an application
  • Bar expands to show detailed information about the application
  • The expanded content shows tabbed panes for displaying an Overview and Document types
  • To the right of the panes is a section for Keywords, Next actions and Activity
  • A ‘Review case’ button links to a documents page

Title: Round 3 (COA)  (View)

Changes:
  • Tabbed view is replaced by a single view to allow Overview and Documents to be seen simultaneously
  • Next action is moved to a more prominent position and highlighted with a blue box
  • Split buttons are used to displays a dropdown with a set of choices for either the application as a whole or the document
  • Selecting ‘Rename’ will display a modal for entering a document name
  • A ‘Done’ button is added to indicate when changes are ready to be submitted

Title: Round 3.1 (COA)  (View)

Changes:
  • Split buttons are replaced with links, making all possible actions visible from the first window
  • ‘Done’ button is shown in a disabled state (only after an action has been performed will the button become active)

Title: Round 4 (COA)  (View)

Changes:
  • Overview removed to ensure that all focus is on the required actions
  • Descriptive language used to increase clarity of instructions
  • Notes added to allow users to record reasons behind decisions
  • Increased differentiation between documents for review and other application documents
